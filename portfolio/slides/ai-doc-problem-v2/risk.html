리스크 검토

선정한 도구·방식에 대해 제기될 수 있는 우려를 미리 정리하고, 각각에 대한 대응 방안을 준비했다.

Q

Claude Code 사용 비용이 부담되지 않는가?

A

백엔드·프론트 개발과 달리 간단한 마크업 문법 수준의 작업이라 코드 양이 많지 않다. html·js·css로 구분해 모듈화하고 반복 사용되는 요소를 공용 컴포넌트로 만들면 AI가 읽어야 할 토큰도 크게 절감할 수 있다. 무엇보다 작업 시간 단축으로 얻는 비용 절감 효과가 도구 사용 비용을 상회한다.

Q

디자이너·사업 담당자는 Claude Code / GitHub에 접근하기 어렵지 않은가?

A

작성·검토는 기획자가 담당하고, 최종 산출물은 PDF처럼 접근이 쉬운 형태로 렌더링해 공유한다. 비개발 직군은 결과 공유 단계에서만 접근하면 되므로 도구 진입 장벽이 업무를 막지 않는다.

Q

AI가 생성·검토한 내용을 그대로 믿어도 되는가?

A

AI는 초안 작성과 검토를 보조할 뿐, 최종 확정은 기획자가 판단한다. 다만 반복적인 교차 점검을 자동화함으로써 사람이 놓치기 쉬운 누락과 모순을 오히려 줄여준다.

Q

Claude Code보다 더 좋은 AI 모델이 나와 도구를 교체하게 되면 작업이 손실되지 않는가?

A

산출물은 GitHub 저장소에 관리되므로 도구를 교체해도 데이터와 변경 이력은 그대로 유지된다. Claude Code는 작성·검토 도구일 뿐 저장소가 아니기 때문이다.
