해결방안

세 가설을 공용 컴포넌트 · 바이브코딩 · 통합 저장 · AI 검토라는 하나의 워크플로우로 엮어 해결했다.

가설 1 — 공용 컴포넌트로 작성 시간 단축
1

공용 컴포넌트 자동 추출

Claude Code가 프로젝트 코드를 분석해 반복 UI 패턴을 컴포넌트 단위로 정리. 실제 구현과 일치하는 기준 자산 확보.

2

컴포넌트 명령으로 와이어프레임 작성

"상단에 GNB OO type"처럼 컴포넌트 이름을 조합한 명령만으로 AI가 화면 생성. 기획자는 로직·구성에 집중.

가설 2 — 동작 프로토타입으로 소통 비용 감소
1

바이브코딩 동작 프로토타입

정적 화면 대신 클릭·입력·전환이 가능한 실행형 프로토타입 생성. 개발 전 단계에서 실물에 가까운 검증.

2

인터랙션 흐름 시연

문서로 설명하기 어려운 동작·전환을 프로토타입이 직접 보여줌. 재확인 질문과 소통 오류 감소.

가설 3 — 통합 저장 + AI 정합성 검토
1

단일 저장소 통합 저장

GitHub 한 저장소에 기획·프로토타입·가이드 문서를 모두 통합. 변경 이력이 맥락으로 누적.

2

유지보수 규약 .md 코드화

Claude Code 명령어·작성 규칙·유지보수 가이드를 CLAUDE.md 등 파일로 저장소에 명문화.

3

AI 정합성 자동 검토

GitHub에 푸시할 때마다 Claude Code가 자동 실행되어 관련 문서 간 모순·누락을 검토. 변경 시점마다 상시 검증.
