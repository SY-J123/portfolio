UX 가이드라인 제작

UX Planning · In Progress

모호하게 사용되던 UX 기준을 실행 가능한 언어로 정리하고,
평가 프레임워크와 제작 가이드라인으로 연결한 프로젝트입니다.

Summary

기획과 디자인이 같은 기준으로 UX를 판단할 수 있도록 정리한 가이드라인 프로젝트.

Point 1
편하게 만들어야 한다는 식의 모호한 표현을 구체화하고 정량화할 필요가 있었다.
Point 2
UX를 일관된 기준으로 판단하기 위한 평가 프레임워크를 만들었다.
Point 3
실제 제작 과정에서 참고할 수 있는 UX 가이드라인을 정리했다.
Point 4
디자이너와 협업해 심미성처럼 기획자 혼자 정의하기 어려운 기준까지 함께 문서화했다.

Background

실무에서 UX에 대해 이야기할 때 “편하다”, “직관적이다”, “보기 좋다” 같은 표현은 자주 등장하지만, 실제 제작과 리뷰 단계에서는 각자 다른 기준으로 해석되는 경우가 많았다. 그래서 기획과 디자인, 리뷰 과정에서 모두가 참고할 수 있는 공통 기준을 문서화할 필요가 있었다.

Problem

모호한 표현은 회의 중에는 쉽게 합의된 것처럼 보이지만, 막상 화면을 만들고 검토할 때는 판단 기준이 달라 반복 수정으로 이어지기 쉬웠다.

  • “편하다” 같은 표현이 구체적인 설계 기준으로 연결되지 않았다.
  • 리뷰 시점마다 다른 관점이 개입돼 일관된 평가가 어려웠다.
  • 심미성 기준은 특히 기획자 혼자 정리하기 어려운 영역이었다.

Goals

  • 추상적인 UX 표현을 실제 제작과 리뷰에 사용할 수 있는 기준으로 바꾼다.
  • 화면 평가에 활용할 수 있는 프레임워크를 정리한다.
  • 기획과 디자인이 함께 참고할 수 있는 제작 가이드라인을 만든다.
  • 심미성과 사용성을 함께 보는 협업 기준을 정리한다.

Approach

1. 모호한 표현을 판단 가능한 기준으로 바꿨다

“편하다”, “복잡하지 않다” 같은 표현이 실제로 어떤 상태를 의미하는지 쪼개고, 가능한 항목은 더 구체적인 체크 기준이나 정량 관점으로 연결했다.

2. 평가 프레임워크를 만들었다

UX를 감각적으로만 이야기하지 않도록, 화면과 흐름을 일정한 기준으로 볼 수 있는 평가 프레임워크를 정리했다.

3. 제작 단계용 UX 가이드라인으로 연결했다

평가만 가능한 문서가 아니라, 실제 설계와 제작 과정에서 미리 참고할 수 있도록 UX 가이드라인 형태로 정리했다.

4. 디자이너와 함께 심미성 기준까지 확장했다

기획자 혼자 판단하기 어려운 시각적 완성도 영역은 디자이너와 협업해 기준을 함께 정리함으로써 더 현실적인 가이드라인으로 발전시켰다.

Evaluation Framework

추상적인 인상을 그대로 두지 않고, UX를 어떤 축으로 나눠 판단할지 먼저 정의했다. 아래 구조는 실제 리뷰와 제작 기준으로 사용할 수 있는 상위 프레임이다.

Top-Level

7개의 동등한 평가 주제

각 주제는 병렬 관계이며, 아래 항목은 해당 주제를 판단하기 위한 체크 포인트다.

01

유용성

  • 사용자의 주요 Task와 직접 연결되는가
  • 실제 문제 해결에 기여하는가
  • 있으면 좋은 수준의 기능에 머물지 않는가

02

사용성

  • 학습 용이성: 처음 써도 이해 가능한가
  • 효율성: 적은 단계로 목표를 달성할 수 있는가
  • 오류 방지: 실수를 줄일 수 있는가
  • 회복 가능성: 실수 후 복구할 수 있는가

03

접근성

  • 색상 대비와 폰트 크기가 충분한가
  • 스크린리더 사용을 고려했는가
  • 터치 영역이 충분히 확보되어 있는가

04

일관성

  • 동일 기능이 동일 UI로 표현되는가
  • 버튼 위치와 인터랙션이 예측 가능한가
  • 플랫폼 가이드를 벗어나지 않는가

05

신뢰성

  • 에러 메시지가 명확한가
  • 데이터 처리 방식이 투명한가
  • 보안과 개인정보 측면에서 신뢰를 주는가

06

감성/매력

  • 비주얼 완성도가 충분한가
  • 마이크로 인터랙션이 경험을 돕는가
  • 브랜드 톤앤매너가 일관되게 느껴지는가

07

성능

  • 로딩 속도가 충분히 빠른가
  • 반응 속도가 지연 없이 느껴지는가
  • 크래시나 불안정성이 없는가

What I Learned

UX 가이드라인은 단지 좋은 원칙을 모아두는 문서가 아니라, 팀이 같은 기준으로 만들고 검토하기 위한 언어를 정리하는 작업이라는 점을 다시 확인했다. 특히 사용성과 심미성은 분리된 영역이 아니라, 서로 다른 전문성을 가진 사람들이 함께 기준을 다듬을 때 더 실무적인 형태가 된다는 점이 중요했다.

Outcome

  • UX 판단 기준을 더 구체적인 언어로 정리하는 초안을 만들었다.
  • 평가 프레임워크와 제작 가이드라인의 기본 구조를 수립했다.
  • 디자인 협업을 포함한 심미성 기준 정리 범위를 확장했다.
  • 세부 지표와 적용 사례는 추후 보완 예정이다.
