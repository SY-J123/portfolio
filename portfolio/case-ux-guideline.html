UX 가이드라인 제작

LULU.AI · 26년 1월 ~ 2월

모호하게 사용되던 UX 기준을 실행 가능한 언어로 정리하고,
평가 프레임워크와 제작 가이드라인으로 연결한 프로젝트입니다.

Summary

모호한 UX 기준을 더 빠르게 판단하고 활용할 수 있는 구조로 정리한 프로젝트.

Problem
기능과 배리언트 기준이 모호해 같은 UI도 사람마다 다르게 이해하고 반복적으로 확인해야 했다.
Goal
UX 판단 기준을 일관되게 정리해 기획과 디자인이 같은 기준으로 빠르게 협업할 수 있도록 만든다.
How
기능 정의서, AI 기반 시각화, 의사결정 트리와 체크리스트를 함께 구성해 실무에서 바로 참고할 수 있게 했다.
Outcome
평가 프레임워크와 UX 가이드라인을 정리해 반복적인 판단 비용을 줄일 수 있는 기준을 만들었다.

Background

화면을 설계하고 협업하는 과정에서 각 UI가 어떤 목적을 가지고 있고, 어떤 기능과 배리언트를 포함하는지 명확하게 정리할 필요가 있었다. 또한 실제 작업 중 빠르게 판단할 수 있는 의사결정 구조가 따로 정리되어 있지 않아, 문서와 함께 이를 보완할 필요가 있었다.

Problem

기능과 배리언트에 대한 기준이 명확하지 않으면, 같은 UI도 사람마다 다르게 이해하거나 다른 방식으로 설계할 수 있었다. 그 결과 화면별 정의가 일관되지 않게 쌓이고, 작업 중에는 매번 비슷한 내용을 다시 확인하거나 논의해야 했다. 또한 빠르게 참고할 수 있는 의사결정 구조가 없어서, 어떤 UI를 선택해야 하는지 판단하는 과정에서도 반복적인 커뮤니케이션 비용이 발생했다.

  • 같은 UI도 사람마다 다르게 이해하거나 설계될 수 있었다.
  • 화면별 기능 정의와 구성이 일관되지 않게 쌓일 수 있었다.
  • 빠르게 참고할 수 있는 의사결정 구조가 없어 반복적인 확인과 커뮤니케이션이 필요했다.

Goals

  • 일관된 기준으로 UI 기능과 배리언트를 체계적으로 정리할 수 있는 문서 구조를 만든다.
  • 문서를 빠르게 이해할 수 있도록 시각적으로 정리된 형태로 발전시킨다.
  • 의사결정 트리와 체크리스트를 통해 빠른 판단 기준을 제공한다.

Approach

1. UI 기능 정의서를 작성했다

각 기능의 목적과 포함되는 요소, 그리고 필요한 데이터와 타입을 구조적으로 정리해 UI를 기능 단위로 이해할 수 있도록 했다.

2. 문서를 AI로 시각화했다

텍스트 중심의 문서를 AI를 활용해 시각화함으로써, 복잡한 내용을 더 빠르게 이해하고 공유할 수 있는 형태로 정리했다.

3. 의사결정 트리와 체크리스트를 작성했다

실제 작업 중 빠르게 판단할 수 있도록, 반복적인 논의를 줄이기 위한 의사결정 트리와 체크리스트를 함께 정리했다.

Outcome

  • UI 기능 14종을 정리했다. (모달, GNB 등)
  • AI를 활용해 시각화 자료와 설명을 함께 확인할 수 있는 HTML 문서 1종을 제작했다.
  • 의사결정 트리와 체크리스트 세트를 정리했다.

What I Learned

문서의 완성도는 정보의 양보다도, 얼마나 빠르게 이해되고 의사결정에 활용될 수 있는지에 달려 있다는 점을 배웠다. 기능 정의와 시각화, 판단 기준을 함께 설계했을 때 문서가 실제 협업 도구로 기능할 수 있었다.
