UX Guideline Development

UX Planning · In Progress

A project to turn vague UX language into executable criteria,
then connect it to an evaluation framework and production guidelines.

Summary

A guideline project that helps planners and designers evaluate UX with shared criteria.

Point 1
Vague statements such as “make it easy to use” needed to be translated into clearer and more measurable criteria.
Point 2
I created an evaluation framework to assess UX with more consistent judgment.
Point 3
I organized UX guidelines that could be referenced during actual production.
Point 4

Background

Teams often use UX terms such as “easy,” “intuitive,” or “visually clean,” but those phrases are interpreted differently during production and review. This project started from the need to document a shared set of criteria that both planning and design could rely on.

Problem

Ambiguous language may sound aligned in meetings, but it often breaks down when screens are actually designed and reviewed, leading to repeated revisions.

  • Expressions like “easy to use” were not directly usable as design criteria.
  • Review outcomes changed depending on who was evaluating the work.
  • Aesthetic quality was especially difficult for planners to define on their own.

Goals

  • Turn abstract UX language into criteria that can guide production and review.
  • Create a framework for evaluating screens and flows more consistently.
  • Build practical UX guidelines that both planners and designers can use.
  • Align usability and aesthetics within a shared collaboration standard.

Approach

1. I translated vague language into judgeable criteria

I broke broad phrases such as “comfortable” or “not complex” into more specific conditions, and where possible connected them to clearer checkpoints or measurable dimensions.

2. I created an evaluation framework

To avoid discussing UX only as a matter of feeling, I organized a framework that could be used to review screens and flows through a more stable lens.

3. I connected it to production-stage guidelines

The document was not only for evaluation after the fact, but also designed to guide decision-making during planning and production.

4. I extended the standard through designer collaboration

For visual quality, which is harder for planners to define independently, I worked with a designer so the guideline could reflect more practical aesthetic standards.

Evaluation Framework

Rather than leaving UX as a vague impression, I defined the top-level criteria first. This structure works as a practical framework for both review and production decisions.

Top-Level

Seven Equal Evaluation Themes

Each theme sits at the same level, and the bullets below are the review criteria for that theme.

01

Usefulness

  • Does it connect to a core user task?
  • Does it solve a real user problem?
  • Is it more than just a nice-to-have?

02

Usability

  • Learnability: is it understandable for first-time users?
  • Efficiency: can users reach the goal in fewer steps?
  • Error prevention: can mistakes be reduced?
  • Recovery: can users recover after mistakes?

03

Accessibility

  • Are contrast and font size sufficient?
  • Is screen reader use considered?
  • Are touch targets large enough?

04

Consistency

  • Is the same function shown with the same UI?
  • Are button positions and interactions predictable?
  • Does it follow platform conventions?

05

Credibility

  • Are error messages clear?
  • Is data handling transparent?
  • Does it feel trustworthy in security and privacy?

06

Desirability

  • Is the visual quality high enough?
  • Do micro-interactions improve the experience?
  • Is the brand tone applied consistently?

07

Performance

  • Is loading fast enough?
  • Does the interface respond quickly?
  • Is the product stable without crashes?

What I Learned

This project reinforced that UX guidelines are not just a collection of good principles. They are a way to give teams a shared language for making and reviewing work. It also showed that usability and aesthetics become far more actionable when people with different expertise shape the criteria together.

Outcome

  • Drafted a clearer language system for UX judgment.
  • Established the base structure for an evaluation framework and production guideline.
  • Expanded the scope to include aesthetic standards through design collaboration.
  • Detailed metrics and applied examples will be added later.
