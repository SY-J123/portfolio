Situation

제한된 시간 안에 여러 산출물을 병행 작성해야 해 각 문서의 기획 의도를 충분히 구체화하기 어려웠다.

기획 문서가 여러 협업 도구에 분산되어 있어 문서별 내용의 불일치가 존재했다.

기획 산출물에 담긴 작업 기준이 충분히 구체적이지 않아 작업자가 구현 기준을 한 번에 파악하기 어려웠다.

기획 문서가 여러 협업 도구에 분산되어 있어 문서별 내용의 불일치가 존재했다.

Pain Point

산출물마다 기획 완성도가 고르지 못해 후속 작업자가 기준을 해석하거나 보완해야 하는 지점이 생겼다.

작은 변경에도 여러 산출물을 함께 수정해야 해 내용 충돌이 쉽게 발생했다.

기획의 명확성이 떨어져 반복적인 재확인으로 작업 속도가 저하되고, 자의적인 해석으로 기획 의도와 작업 결과의 불일치가 발생했다.

문서마다 내용이 조금씩 달라 어떤 내용을 기준으로 구현해야 하는지 혼선이 생겼다.

Needs

정확한 기획 의도를 충분히 담아 후속 작업자에게 전달하고 싶다.

변경이 생겨도 한 번에 정합성을 유지할 수 있는 구조가 필요하다.

더 명확하게 정리된 기획 내용을 전달받고 싶다.

문서의 정합성이 유지되어 재확인 없이 구현하고 싶다.
