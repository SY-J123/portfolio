# VOC 자동수집 대시보드 — 서비스 흐름도

## 1. 데이터 수집 → 분석 파이프라인

```
[VOC 소스]                [수집]              [AI 분석]            [스코어링]          [시각화]
 
 앱스토어 리뷰  ──┐
 플레이스토어    ──┤
 커뮤니티 글    ──┼──→  자동 크롤링  ──→  Claude API  ──→  점수 산출   ──→  대시보드
 CS 인입       ──┤      (스케줄링)       ├ 감정 분석       ├ Sentiment      ├ 종합 스코어
 SNS 멘션      ──┘                       ├ 카테고리 분류    ├ Severity       ├ 카테고리 점수
                                         ├ 심각도 판단     └ Score_reality  ├ 추이 그래프
                                         └ 품질 속성 매핑                    └ VOC 피드
```

## 3. 프로젝트 소개 탭 구성안 (포폴용)

```
프로젝트 소개 탭
│
├── 프로젝트 개요
│   ├── 한 줄 요약: "VOC를 자동 수집·분석하여 서비스 품질을 점수화하는 대시보드"
│   ├── 배경: 수작업 VOC 모니터링의 한계 → 자동화 필요성
│   └── 목표: VOC 기반 의사결정 속도 향상
│
├── 문제 정의
│   ├── AS-IS: VOC 산재, 수작업 분류, 주관적 판단
│   └── TO-BE: 자동 수집 → AI 분류 → 정량 스코어링 → 시각화
│
├── 나의 역할
│   ├── 서비스 기획: IA 설계, 화면 흐름 정의, 분석 기준 수립
│   ├── 데이터 설계: 29개 품질 속성 정의, 스코어링 공식 설계
│   └── 프론트엔드: Next.js 기반 대시보드 구현
│
├── 기술 스택
│   ├── Frontend: Next.js, Vercel
│   ├── AI: Claude API (감정 분석, 카테고리 분류)
│   └── Data: 자동 크롤링 파이프라인
│
└── 주요 성과
    ├── VOC 분류 자동화율: 수작업 → 100% 자동
    ├── 모니터링 소요 시간: 기존 대비 N% 단축
    └── 4개 축 품질 스코어링 체계 구축
```
