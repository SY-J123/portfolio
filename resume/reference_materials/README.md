## 파일별 참고 포인트

### 1. `naverwebtoon_ui_designer_reference.pdf`
- 자소서 문항에 대해 강점을 설명할 때 프로젝트 사례를 바로 연결하는 방식 참고
- `리서치`, `프로토타이핑`, `커뮤니케이션`처럼 본인 강점을 업무 언어로 풀어내는 방식 참고
- 다만 문장 단위는 길고 서술형 비중이 커서, 구조만 참고하고 밀도는 더 높게 가져가는 편이 좋음

### 2. `hyundai_self_intro_reference.pdf`
- `지원 동기`, `직무 강점`처럼 문항이 직접적인 자소서 구조 참고
- 강점을 말할 때도 실제 프로젝트 역할과 결과를 붙이는 방식 참고
- `어떤 프로젝트에서 어떤 요소를 담당했는지`를 분명히 드러내는 방식 참고

### 3. `alive_cv_reference.pdf`
- 가장 직접적인 레퍼런스
- `경력사항`과 `프로젝트 수행이력`을 분리하고, 역할을 업무 단위로 상세하게 적는 방식 참고
- `리딩`, `조사/분석`, `기획`, `운영`, `지표 관리`처럼 역할을 명확하게 쓰는 방식 참고
- 우리 자소서 2번과 3번에서 `역할`, `업무`, `지표`를 더 구체화할 때 가장 유용한 레퍼런스

## 이번 작업에서 얻은 핵심 인사이트
- 추상적인 자기정의보다 `무슨 일을 맡았는지`가 먼저 보여야 함
- `리딩`, `리서치`, `설계`, `운영`, `분석`, `조율`처럼 역할의 결이 보여야 함
- 강점도 성향이 아니라 `업무 + 결과`로 증명하는 편이 설득력이 높음
- 2번 `주요 경력`은 경력 흐름 설명만으로 끝내지 말고, 주요 역할과 짧은 성과까지 보여줘야 함
- 3번은 대표 프로젝트를 깊게 설명하는 구조가 더 적절함
