# 지마켓 Transaction조직 서비스기획자 자기소개서

## 1. 지원 동기 및 이직 사유

저는 기능 설계에 머무르지 않고, 실제 서비스 안에서 문제를 정의하고 개선 방향을 검증할 수 있는 기획자가 되고 싶습니다. 좋은 기획은 문서를 만드는 것에서 끝나는 것이 아니라, 그 결과를 더 정확하게 검토하고 더 나은 의사결정으로 연결하는 과정까지 포함한다고 생각합니다. 그래서 사용자 경험과 도메인 이해뿐 아니라, 데이터와 정량 판단 역량도 함께 보완해 왔습니다.

지마켓 Transaction조직은 주문, 결제, 장바구니라는 구매 핵심 동선을 직접 설계하고 개선하는 조직입니다. 사용자의 구매 전환에 직접 영향을 미치는 서비스 정책을 수립하고, 데이터를 기반으로 흐름을 고도화하는 업무는 제가 쌓아온 서비스 정책 설계, 사용자 흐름 기획, 운영 데이터 기반 개선 경험과 맞닿아 있다고 판단했습니다.

## 2. 주요 경력 및 강점

저는 운영과 기획을 모두 경험하며, 서비스가 실제로 어떻게 작동하는지 이해한 상태에서 문제를 정의하고 실행 기준으로 구체화하는 역량을 쌓아왔습니다. 미투온과 앤서스랩코리아에서는 운영 현황과 지표를 점검하고, 해외 지사 및 파트너와 요구사항을 조율하며 서비스가 돌아가는 실제 흐름과 제약을 익혔습니다. 특히 전년도 손실이 컸던 운영 건에서는 상금 구조와 스케줄, 주요 지표를 다시 점검해 더 적합한 운영안을 제안했고, 그 결과 손실 폭을 절반 수준으로 줄였습니다.

이후 나트리스에서는 기능 기획, UX 설계, 서비스 정책 수립 업무를 맡으며 문제를 실제 실행 구조로 바꾸는 역할을 담당했습니다. 복잡한 비즈니스 로직을 기능 정의서와 예외 흐름으로 구체화하고, 사용자 흐름 기반으로 화면 전환 구조를 설계하는 과정에서 서비스 정책과 기능 기획을 End-to-End로 수행하는 경험을 쌓았습니다. 요구사항이 모호하거나 업무 경계가 불분명한 상황에서도, 무엇을 문서로 정의하고 무엇을 협의로 넘길지 먼저 정리해 여러 직군이 같은 기준 위에서 움직일 수 있도록 구조를 만드는 데 강점이 있습니다.

## 3. 협업 방식

저는 협업 도구 자체보다, 팀이 어떤 기준으로 더 빠르고 정확하게 움직일 수 있는지가 중요하다고 생각합니다. AX 프로젝트에서는 초기 보고와 검토 단계마다 여러 직군이 함께 투입되면서 준비 시간과 소통 비용이 반복적으로 커지는 문제가 있었습니다. 저는 이를 단순한 문서 작성 이슈가 아니라 협업 구조의 문제라고 보고, 기획자가 직접 바이브 코딩으로 초기 프로토타입을 제작하는 방식으로 검토 구조를 다시 설계했습니다. 그 결과 이해관계자 투입을 3명에서 1명으로 줄였고, 보고 및 컨펌 리드타임도 1주일에서 3일로 단축했습니다.

또한 GitHub 기반 문서 관리 방식을 도입해, 비개발 직군도 쉽게 접근할 수 있도록 웹에 배포된 문서를 최신본 기준으로 삼는 구조로 전환했습니다. 그 결과 버전 이력과 최신 기준을 함께 관리하면서도, 어떤 문서가 기준인지 다시 확인하는 반복 커뮤니케이션을 줄일 수 있었습니다.

## 4. 입사 후 계획

입사 후에는 먼저 G마켓/옥션의 주문·결제·장바구니 서비스 구조와 정책을 빠르게 파악하고, 사용자 구매 동선에서 전환율에 영향을 주는 병목 지점을 우선적으로 이해하겠습니다. 그다음 운영 데이터와 사용자 행동을 기반으로 개선 가설을 수립하고, 서비스 정책과 기능 기획으로 연결하는 데 기여하고 싶습니다.

또한 저는 통계학과 데이터 분석 역량을 꾸준히 보완하고 있습니다. 앞으로는 사용자 경험과 기획 감각뿐 아니라 데이터 기반 판단 역량까지 함께 갖춰, Transaction 도메인에서 더 설득력 있는 의사결정과 개선 방향을 제안할 수 있는 기획자로 성장하겠습니다.
