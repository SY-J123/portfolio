## Project 1

### 제목
AI 기반 문서 협업 프로세스 개선

### 헤드라인
문서 중심 협업 구조를 재설계해 핸드오프 준비 시간을 2~3일에서 반나절로 단축

### Role
문서 중심 협업 구조의 문제를 발견하고, AI 도구를 활용한 문서 시스템과 협업 프로세스를 설계/실행

### Background
- 서비스: B2C 모바일 웹 텍사스 홀덤
- 팀 규모: 약 10명
- 기획자: 2명
- 조직 특징: POC 중심, 빠른 기능 검증/실험 중시

### Problem
- 기능 기획 문서 작성에 많은 시간 소요
- 작업 순차 진행으로 대기 시간 발생
- 작은 수정에도 문서 수정 -> 전달 -> 재작업 반복
- 텍스트 중심 문서로 UX 전달 어려움
- 회의에서 서로 다른 화면을 상상
- 문서 작업 비중이 높아 UX/제품 가치 고민 시간 부족

### Attempt 1 (Notion AI)
목표
- 문서 자동 정리
- 일정한 포맷 문서 생성

결과
- 문서 포맷 정리
- 유지보수 일부 개선

한계
- 텍스트 중심 문서 피로 지속
- UX 전달 문제 미해결
- AI가 전체 문서 맥락 이해 어려움

### Solution
문서 작성 방식이 아니라 협업 구조 자체를 재설계

1. GitHub 문서 구조
- 모든 문서를 Markdown으로 통합
- 이유: AI 코드 도구 연동 유리, 버전 관리, 히스토리 추적, 탐색성 개선

2. 문서 역할 분리
- 요구사항 문서: Markdown
- 핸드오프 문서: HTML (시각 자료 중심)

3. 프로토타입 제작
- 바이브 코딩 방식으로 프로토타입 제작
- 설계 문서 + 핸드오프 문서 + 프로토타입 구조로 협업 개선

### Impact
- 문서 초안 작성 시간: 반나절 -> 약 2시간 (약 60% 감소)
- 핸드오프 준비 시간: 2~3일 -> 반나절 (약 75% 감소)

추가 변화
- 논의 속도 개선
- 수정 횟수 감소
- 문서 탐색성 개선
- 문서 무결성 관리 개선
- 기획자가 UX/제품 가치에 더 집중 가능

### Insight
AI가 문서를 작성할 수 있는 환경에서는,
기획자의 역할은 문서를 작성하는 일에서 제품 구조 설계와 문서 품질 검토로 이동한다.
기획자는 제품 가치, UX, 기능 구조 같은 본질 문제에 더 집중해야 한다.
