# VOC 분류 가이드라인

이 문서는 Google Play 리뷰 VOC를 일관된 기준으로 분류하기 위한 규칙이다.
LLM(Claude Code)이 여러 턴에 걸쳐 batch 분류할 때 매번 이 문서를 참조한다.

---

## 1. 입출력 포맷

### Input
`data/google-play.march-plus.json` — 3월 1일 이후 전처리된 리뷰 배열.

### Output
`data/google-play.classified.json` — 입력에 `classification` 필드를 추가한 배열.
분류 대상에서 **제외된 리뷰(광고·스팸)는 결과에 포함하지 않는다.**

```json
{
  "source": "google_play",
  "external_id": "...",
  "author": "...",
  "score": 1,
  "text": "...",
  "posted_at": "...",
  "app_version": "...",
  "thumbs_up": 0,
  "classification": {
    "quality_attributes": ["알림 적절성", "고객지원 접근성"],
    "sentiment": "부정",
    "sentiment_score": -10,
    "severity": "반복/누적",
    "severity_multiplier": 1.5,
    "score_reality": 65
  }
}
```

`score_reality = 80 + (sentiment_score × severity_multiplier)` 후 0~100 clip.

---

## 2. 품질 속성 (quality_attributes)

29개 속성 중 해당되는 것을 배열로 반환. **multi-label 허용, 최대 3개.**

해당 속성 없으면 `[]`. 전략·UX·운영·기술 4개 카테고리 아래 속성:

### 전략 (7개)
- `핵심가치전달력` — 토스가 뭐가 좋은지 모르겠음
- `차별화 인식` — 카뱅/토뱅 등 경쟁사 비교
- `시장 적합성` — 특정 타겟층에 안 맞음
- `가격 대비 가치` — 유료·프라임·혜택 언급
- `정책 투명성` — 수수료·요금·가격 정책
- `프로모션 체감` — 쿠폰·포인트·이벤트 혜택
- `타겟 적합성` — 어르신·특정 세대 배제

### UX (8개)
- `UI 일관성` — 화면별 디자인·인터랙션 통일성
- `정보구조 적합성` — 메뉴·카테고리·탐색
- `온보딩 경험` — 최초 사용·시작 방법
- `검색 효율성` — 검색 기능·결과
- `접근성` — 글자 크기·색약·확대
- `오류 메시지 품질` — 에러 메시지 이해도
- `이용 내역 접근성` — 거래·이체 내역 다시 찾기
- `시각적 피로도` — 다크모드·밝기·눈피로

### 운영 (7개)
- `고객지원 접근성` — 고객센터·상담·전화·문의 채널
- `응답 속도 체감` — 답변 지연
- `처리 정확성` — 요청 오처리
- `공지·안내 명확성` — 정책 변경·업데이트 공지
- `후기 신뢰도` — 리뷰 조작·별점 관리
- `알림 적절성` — 푸시 빈도·내용
- `거래 상태 안내` — 송금·이체 상태

### 기술 (7개)
- `크래시 빈도` — 튕김·꺼짐·비정상 종료
- `앱 안정성` — 먹통·반응 없음·업데이트 후 불안정
- `배터리 영향` — 배터리 소모
- `로딩 속도` — 화면 전환·데이터 로딩 지연
- `로그인·인증` — 로그인·본인인증·OTP
- `보안·개인정보` — 유출·해킹·개인정보
- `호환성` — 기기·OS·태블릿 레이아웃

### 할당 규칙
- **multi-label**: 한 리뷰가 여러 속성에 걸치면 해당되는 속성 모두 뽑되 **최대 3개**
- **우선순위**: 본문에서 가장 구체적으로 서술된 것부터
- **무속성 감정 표현** ("최악이에요", "좋아요"만 있는 경우): `[]`
- **광고·스팸**: 결과에서 **제외** (분류 안 함)

## 3. 감정 극성 (sentiment)

### 점수
| 단계 | sentiment_score |
|---|---|
| 매우 긍정 | +20 |
| 긍정 | +10 |
| 중립 | 0 |
| 부정 | −10 |
| 매우 부정 | −20 |

### 판정 순서
1. **별점 baseline**:
   - 5점 → 긍정 (+10)
   - 4점 → 긍정 (+10)
   - 3점 → 중립 (0)
   - 2점 → 부정 (−10)
   - 1점 → 부정 (−10)
   - 없음 → 텍스트 단독 판정

2. **텍스트 신호로 ±1단 보정**:
   - **매우 긍정 트리거**: 없으면 안 됨 / 혁신 / 최고 / 대체 불가 / 감탄 표현
   - **매우 부정 트리거**: 신뢰 훼손 / 금전 피해 / 이탈 의지 (다신 안 씀, 삭제, 사기, 최악, 불매, 해지)

3. **충돌 시 텍스트 우선**:
   - 별점 5인데 본문이 불만이면 → 부정
   - 별점 1인데 본문이 감사·칭찬이면 → 긍정
   - (이유: 별점 실수가 텍스트보다 흔함)

4. **질문만 있는 리뷰**: 중립
5. **칭찬+불만 혼재**: 본문 무게중심 우세 쪽

---

## 4. 심각도 (severity)

### 배수
| 단계 | severity_multiplier |
|---|---|
| 일반 | ×1.0 |
| 반복/누적 | ×1.5 |
| 이탈/강한 부정 | ×2.0 |
| 반복+이탈 | ×2.5 |

### 판정 규칙
- **반복 트리거** (있으면 반복/누적 이상):
  `또 | 계속 | 자꾸 | 매번 | N번째 | 이번 주만 | 항상 | 반복 | 여러 번 | 맨날`
- **이탈 트리거** (있으면 이탈/강한 부정 이상):
  `삭제 | 지움 | 탈퇴 | 넘어가 | 해지 | 안 씀 | 안 쓴다 | 불매 | 신고 | 사기 | 최악 | 다신 안 | 못 믿`
- **둘 다** 있으면 **반복+이탈 (×2.5)**
- **반복만** → 반복/누적 (×1.5)
- **이탈만** → 이탈/강한 부정 (×2.0)
- **둘 다 없음** → 일반 (×1.0)
- **긍정 리뷰는 무조건 ×1.0** (감정이 양수이면 severity 고려 불필요)

---

## 5. 제외 대상 (광고·스팸)

다음에 해당하면 **결과에서 완전히 제외**:
- 다른 앱·서비스·상품 광고
- 외부 링크·이메일·연락처 유도
- 의미 없는 반복 문자열 (이미 전처리에서 상당수 걸러짐)
- 토스와 무관한 내용

단순히 짧거나 평범한 리뷰는 **제외 아님** — 속성 `[]`로 두되 감정·심각도는 판정.

---

## 6. Edge Cases

- **답변(`reply`) 본문은 분류에 사용하지 않음.** 리뷰 본문(`text`)만 본다.
- **이모지만 있는 리뷰**는 전처리에서 걸러졌지만 혹시 남아 있으면 → 중립·무속성 (제외 아님).
- **욕설·과격어**는 매우 부정으로 분류하되, 광고가 아니면 제외 X.
- **토스 하위 서비스 (토스뱅크·토스증권·토스쇼핑·토스프라임) 관련 리뷰**는 토스 VOC로 포함.

---

## 7. 배치 작업 규칙

- 입력: `data/google-play.march-plus.json`의 N번째 ~ M번째 리뷰
- 출력: `data/google-play.classified.json`에 append
- 한 batch에 **30~50건**, 너무 많으면 품질 저하
- 매 batch 시작 시 이 문서를 먼저 재독
- 배치 끝나면 처리한 범위(인덱스)를 출력
